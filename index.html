Figure 1 : Interface de notation et de commentaires du site e-commerce Cablissimo

La Figure 1 est une capture d’écran de l’interface client du site e-commerce Cablissimo, que j’ai développé en équipe dans le cadre d’un projet web. Ce site a été conçu avec le langage Python et le framework Flask, en respectant les principes de développement d’une application fullstack. Le projet avait pour but de permettre à un client de consulter les détails d’un produit, de lui attribuer une note, de rédiger jusqu’à trois commentaires par article acheté, et de gérer ses propres avis.

L’interface met en œuvre plusieurs fonctionnalités visibles dans les zones encadrées en rouge.
La première zone (en haut) montre la gestion des notes : l’utilisateur peut saisir une note (sur 5), la modifier ou la supprimer. La note moyenne affichée est recalculée automatiquement à chaque changement, grâce à une requête SQL utilisant des fonctions d’agrégation.
La deuxième zone rouge permet de rédiger un commentaire, tout en affichant un rappel des règles métier : 3 commentaires maximum par article acheté. Cette contrainte est vérifiée à la fois dans le code serveur Flask et dans la base de données, afin de garantir l’intégrité.
Enfin, la dernière zone affiche les commentaires existants, avec pour chacun le pseudo de l’auteur, son contenu, la date, et un bouton de suppression s’il s’agit de l’auteur connecté. Cela implique une gestion de session pour restreindre les droits d’action à l’utilisateur concerné.

Pour cela, nous avons conçu un MCD (modèle conceptuel de données) avec Looping, que nous avons ensuite traduit en modèle relationnel. Les principales entités sont Utilisateur, Article, Commande, Note et Commentaire, reliées par des relations et des contraintes d’intégrité référentielle. Par exemple, il n’est possible de noter ou commenter un article que si l’utilisateur l’a réellement commandé.

Le rendu HTML a été généré avec Jinja2, ce qui permet d’adapter dynamiquement l’interface en fonction des données du client connecté. Le tout repose sur une architecture client-serveur, avec des routes Flask qui assurent la logique métier, et une validation des données pour garantir la sécurité de l’application (contre les injections SQL ou les XSS, par exemple).

Ce travail m’a permis de mobiliser plusieurs savoirs : la modélisation des bases de données, la programmation web avec Flask, la sécurité des applications web, et la création d’interfaces dynamiques adaptées aux utilisateurs.

Figure 2 : Configuration d'une exécution paramétrée dans IntelliJ IDEA pour le projet Pauvocodeur

La figure 2 est une capture de l’interface d’IntelliJ IDEA, un environnement de développement intégré (IDE) qui facilite grandement la création et l’exécution de programmes en Java. J’ai utilisé cet outil pour développer un projet nommé “Pauvocodeur”, destiné au traitement numérique de fichiers audio au format .wav.

Dans ce projet, la configuration de l’exécution du programme est un élément clé, notamment parce que le programme attend des paramètres en ligne de commande pour fonctionner correctement. Ces paramètres permettent de spécifier, par exemple, le nom du fichier audio à traiter ainsi que certains réglages numériques, comme un facteur de modification de fréquence.

Dans la partie droite de la figure 2, on distingue clairement la fenêtre de configuration d’exécution d’IntelliJ IDEA, dans laquelle j’ai renseigné la classe principale contenant la méthode main, ainsi que les arguments passés lors du lancement du programme. Ici, les arguments linformatique.wav 2 indiquent que le programme doit ouvrir le fichier nommé linformatique.wav et appliquer un facteur de fréquence égal à 2.

Sur la partie gauche de la figure 2, l’arborescence du projet montre plusieurs fichiers générés automatiquement, notamment des fichiers .wav issus des traitements, ainsi qu’un fichier texte contenant la sortie console du programme. Cette organisation témoigne d’une bonne gestion des fichiers et des résultats dans le système.

Cette configuration dans IntelliJ IDEA facilite la gestion des paramètres d’exécution et le test rapide de différentes options, sans nécessiter de modification du code source. Cela illustre ma maîtrise de l’IDE pour compiler, exécuter et déboguer efficacement un programme Java.

En résumé, la figure 2 met en évidence l’importance de bien configurer l’exécution d’un programme Java dans un IDE, notamment quand celui-ci doit traiter des fichiers locaux et recevoir des arguments en ligne de commande, ce qui est essentiel pour le projet Pauvocodeur.
